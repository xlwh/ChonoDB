## 2026-04-06: 数据写入和查询问题

### 问题描述

在测试ChronoDB的数据写入和查询功能时，发现以下问题：

1. **数据写入成功，但查询返回0个结果**
   - 数据生成脚本成功生成了10万个数据点
   - ChronoDB服务器日志显示数据写入成功，没有错误消息
   - 但是所有查询都返回了0个结果

2. **标签解析问题**
   - `/api/v1/label/__name__/values` 接口返回的指标名称包含了很多奇怪的值
   - 例如：`['191621790924', '2"}', '3', 'ion_0"}', '1834952', ...]`
   - 正确的指标名称应该是 `test_metric`

### 问题分析

#### 数据生成脚本

数据生成脚本生成的数据格式如下：

```
test_metric{job="job_0", instance="instance_0", region="region_0"} 14.205538663556716 1775473340963
```

格式正确，符合Prometheus文本格式。

#### 文本格式解析

`remote_server.rs` 文件中的文本格式数据解析逻辑可能存在问题：

1. 使用 `line.find(' ')` 找到第一个空格，将行分割为指标部分和值时间戳部分
2. 使用 `rest_str.find(' ')` 找到第二个空格，将剩余部分分割为值和时间戳
3. 解析指标名和标签：
   - 如果指标部分包含 `{`，则解析标签
   - 否则，将整个指标部分作为指标名

#### 可能的原因

1. **标签解析问题**：在解析标签时，可能没有正确处理标签字符串，导致标签值被错误地解析
2. **倒排索引问题**：在将标签添加到倒排索引时，可能存在问题
3. **时间戳问题**：`Chunk::add` 方法要求样本的时间戳必须大于当前chunk的最大时间戳，如果时间戳不是递增的，会导致写入失败
